How it works

Five layers of evidence infrastructure

01
Data ingestion
Congress abstract scrapers ingest PDFs and structured data post-embargo. PubMed API feeds published literature. ClinicalTrials.gov tracks ongoing studies.
Vector DB · pgvector
02
Evidence graph
Neo4j knowledge graph maps relationships between trials, guideline updates, and prior literature. Contradiction detection surfaces conflicting evidence.
Neo4j · Knowledge graph
03
RAG pipeline
Query decomposition, semantic retrieval, evidence reranking by study hierarchy. Every claim traces to a source. Hallucination without citation is structurally impossible.
RAG · Reranking
04
LLM synthesis
Claude as reasoning backbone. Structured prompts enforce citation, pre-publication disclosure, conflict flagging, and honest evidence confidence tiers.
Claude API · Structured output
05
Physician interface
Mobile-first React Native, PWA for instant access, and this web portal. Specialty filters, voice input, offline congress digests, push alerts for new trials.
React Native · PWA

Why now

The window
is open

Three forces converge in 2026 that make this the right moment to build Axiom.

i.
The incumbent is US-only
OpenEvidence raised $700M validating physician AI — but serves only US-verified MDs. 400,000 European specialists are entirely unserved.
ii.
Congress data has no owner
Post-embargo public abstracts are reusable for synthesis. No competitor has built the licensing relationships and ingestion infrastructure. First-mover advantage is structurally available.
iii.
AI is now clinically credible
RAG-based citation enforcement has solved hallucination for constrained domains. The EU AI Act (Aug 2025) creates a compliance moat for early registrants.
iv.
CPD obligation = recurring revenue
UK GMC requires 50 CPD hours/year from every licensed physician. Congress synthesis maps to accredited CPD — a non-discretionary, recurring spend.
